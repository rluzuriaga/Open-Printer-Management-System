How does this site work.

Since this website is on a remote server, there aren't any physical printers on the network to use. So I made the demo site accept any IP address and randomly assign a model name and the number/type of toner data.

Click on the Add Printer button to add as many fictitious printers as you'd like.
Clicking the Refresh Toner Data button subtracts 1-3% for each toner on every printer every time it is clicked.
Every hour, on the hour, the system resets to having no printers.


How to produce errors.

The project creates error messages when adding a printer that is off, has no usable SNMP data, or some other unexpected error.
To produce those error messages, I created a set of IP address octets that will always produce the errors. All you need to do is change the last IPv4 address octet to one bellow.

Printer is off:    xxx.xxx.xxx.100
Unexpected error:    xxx.xxx.xxx.200
No valid SNMP data / printer or firmware is too old:    xxx.xxx.xxx.130
Examples: 10.15.155.100, 192.168.45.200

Note: This functionality is only for the demo site. Visit the project's GitHub page to see how the system actually works in a production environment.
